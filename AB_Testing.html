A/B Testing

Statsmodel, python, Numpy, matplotlib, Random and Pandas

INTRODUCTION

A/B tests are very commonly performed by data analysts and data scientists. For this project, I will be working to understand the results of an A/B test run by an e-commerce website. The goal is to work through and analyse the provided data to help the company understand if they should:

  • Implement the new webpage,
  • Keep the old webpage, or
  • Perhaps run the experiment longer to make their decision.

Dataset Description

The company major dataset contains five columns;

  1. "user_id" - Unique ID
  2. "timestamp" - Time stamp when the user visited the webpage
  3. "group" - In the current A/B experiment, the users are categorized into two broad groups. However, some inaccurate rows are
    present in the initial data, such as a "control" group user is matched with a "new_page"
    • "control" - this user group are expected to be served with "old_page"
    • "treatment" - this user group are expected to be matched with th "new_page"
  4. "landing_page" - It denotes whether the user visited the old or new webpage
  5. "converted" - It denotes whether the user decided to pay for the company's product. Here, 1 means yes, the user bought the product.

  3. Given that an individual was in the treatment group, what is the probability they converted?

Part II: A/B Test Approach

Since a timestamp is associated with each event, I could run a hypothesis test continuously as long as I observe the events. However, then the hard questions would be:

  • Do you stop as soon as one page is considered significantly better than another or does it need to happen consistently for a certain amount of time?
  • How long do I run to render a decision that neither page is better than another?

These questions are the difficult parts associated with A/B tests in general. For this project, I considered making my decision based on just all the data provided.

  • Hypothesis Statement

Recall that we just calculated that the "converted" probability (or rate) for the old page is slightly higher than that of the new page. Based on this, the prior assumption to form a basis for the hypothesis statement is that the old page is better unless the new page proves to be definitely better at a Type I error rate of 5%. The Null hypothesis for this project states that the old page(control group) is more effective than the new page while the alternative(competing) hypothesis states that the new page(treatment group) leads to a higher convertion rate amongst the company website visitors. In this final part, you will see that the result achieved in the A/B test in Part II above can also be achieved by performing regression.
